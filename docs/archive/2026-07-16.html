Některé z nich, například ve Firefoxu, již mají zveřejněné exploity. Doporučuje se okamžitá aktualizace všech dotčených systémů a aplikací. Administrátoři jsou vyzýváni k okamžitému aplikování dostupných patchů, aby zabránili kompromitaci systémů. Škodlivé npm balíčky AsyncAPI distribuovaly malware — Pět kompromitovaných verzí balíčků AsyncAPI bylo publikováno do Node Package Manager (npm) v rámci útoku na dodavatelský řetězec. Tyto balíčky obsahovaly trojského koně pro vzdálený přístup s funkcemi pro krádež informací a spuštění botnetu. Vývojáři by měli pečlivě kontrolovat integritu a původ svých závislostí. [Zdroj: BleepingComputer] URL: https://www.bleepingcomputer.com/news/security/-asyncapi-npm-packages-infected-with-credential-stealing-malware/

Nový PoC exploit pro Windows zero-day zveřejněn po Patch Tuesday — Bezpečnostní výzkumník zveřejnil nový proof-of-concept (PoC) exploit s názvem LegacyHive, který zneužívá zranitelnost v User Profile Service systému Windows. K zveřejnění došlo jen několik hodin po vydání pravidelných měsíčních aktualizací Microsoftu, což zdůrazňuje rychlost, s jakou jsou nové zranitelnosti zneužívány. [Zdroj: The Hacker News] URL: https://thehackernews.com/2026/07/researcher-drops-new-windows-zero-day-poc-hours-after-microsoft-patch-tuesday.html

⚠️ Příběh z praxe

Incident: Kompromitace vývojářských balíčků

Co se stalo: Vývojáři stáhli z veřejného repozitáře npm balíčky AsyncAPI, které se zdály být legitimní.

Co udělali útočníci: Útočníci kompromitovali účty legitimních vývojářů nebo repozitáře a nahráli škodlivé verze těchto balíčků, které obsahovaly malware pro krádež přihlašovacích údajů a spuštění botnetu na systémech obětí.

Co oběť zanedbala: Nedostatečná kontrola integrity a původu stahovaných balíčků, spoléhání na důvěru v repozitáře bez dodatečného ověření.

Co měla udělat jinak: Používat nástroje pro skenování závislostí, ověřovat digitální podpisy, kontrolovat reputaci autorů a sledovat bezpečnostní oznámení pro používané knihovny.

🛡️ Praktický tip

Co udělat: Okamžitě aplikujte bezpečnostní aktualizace pro veškerý software, zejména pro Zoom, webové prohlížeče (Firefox, Chrome), Adobe produkty, VMware a Microsoft SharePoint.

Proč: Mnoho kritických zranitelností, včetně těch v Zoom, prohlížečích a SharePointu, je aktivně zneužíváno útočníky. Odkládání aktualizací vystavuje vaši organizaci vysokému riziku kompromitace.

Jak: Nastavte automatické aktualizace, pravidelně kontrolujte dostupnost patchů a prioritizujte nasazení kritických záplat napříč celou infrastrukturou.
