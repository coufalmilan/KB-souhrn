Kritické zranitelnosti v SEPPMail Secure E-Mail Gateway — Byly odhaleny kritické zranitelnosti v podnikovém řešení SEPPMail Secure E-Mail Gateway, které by mohly vést ke vzdálenému spuštění kódu (RCE) a přístupu k e-mailovému provozu. Organizace používající tento produkt by měly neprodleně zkontrolovat dostupnost záplat a provést aktualizaci, aby zabránily kompromitaci e-mailové komunikace. [Zdroj: The Hacker News] URL: https://thehackernews.com/2026/05/seppmail-secure-e-mail-gateway.html

Nové útoky na dodavatelský řetězec přes GitHub Actions a npm — Objevily se nové kampaně útoků na dodavatelský řetězec softwaru, které zneužívají populární GitHub Actions (např. `actions-cool/issues-helper`) a npm balíčky (např. v ekosystému @antv). Útočníci kompromitují vývojářské nástroje a účty, aby vložili škodlivý kód pro krádež přihlašovacích údajů a dat, což představuje vážné riziko pro integritu softwarových projektů. [Zdroj: The Hacker News] URL: https://thehackernews.com/2026/05/github-actions-supply-chain-attack.html

Únik klíčů AWS GovCloud u dodavatele CISA — Dodavatel americké agentury CISA (Cybersecurity & Infrastructure Security Agency) neúmyslně zveřejnil na GitHubu přihlašovací údaje k vysoce privilegovaným účtům AWS GovCloud. Tento incident podtrhuje rizika spojená s nesprávnou správou citlivých informací a přístupových klíčů i u vládních dodavatelů a zdůrazňuje potřebu přísných bezpečnostních auditů. [Zdroj: Krebs on Security] URL: https://krebsonsecurity.com/2026/05/cisa-admin-leaked-aws-govcloud-keys-on-github/

SHub macOS infostealer maskovaný jako aktualizace Apple — Nová varianta infostealeru SHub pro macOS se šíří maskovaná jako falešná bezpečnostní aktualizace od Apple. Po instalaci zobrazí podvodnou zprávu a nainstaluje backdoor, což představuje riziko pro uživatele macOS, kteří by měli být obezřetní vůči nečekaným výzvám k aktualizaci. [Zdroj: BleepingComputer] URL: https://www.bleepingcomputer.com/news/security/shub-macos-infostealer-variant-spoofs-apple-security-updates/

Grafana čelila krádeži zdrojového kódu přes GitHub token — Společnost Grafana Labs oznámila, že útočníci stáhli její zdrojový kód poté, co kompromitovali její prostředí GitHubu pomocí odcizeného přístupového tokenu. Incident zdůrazňuje potřebu robustního zabezpečení vývojových platforem, pečlivé správy přístupových tokenů a monitorování neobvyklých aktivit v repozitářích. [Zdroj: BleepingComputer] URL: https://www.bleepingcomputer.com/news/security/grafana-says-stolen-github-token-let-hackers-steal-codebase/

⚠️ Příběh z praxe

Představte si, že vývojář ve vaší firmě stáhne populární rozšíření pro VS Code nebo použije GitHub Action pro automatizaci CI/CD, které se zdá být legitimní. Útočníci však kompromitovali účet správce tohoto nástroje a vložili do něj škodlivý kód. Vývojář si ničeho nevšimne, ale spuštěním nástroje nevědomky předá útočníkům své přihlašovací údaje nebo přístupové tokeny k firemním repozitářům. Oběť zanedbala ověření integrity a původu nástroje, stejně jako nedostatečné segmentování přístupových práv. Měla by používat striktní politiky pro schvalování externích komponent, pravidelně auditovat používané nástroje a implementovat princip nejmenších privilegií pro vývojářské účty.

🛡️ Praktický tip

Co udělat: Implementujte přísné kontroly a ověřování pro všechny externí knihovny, rozšíření a nástroje používané ve vašem vývojovém prostředí.

Proč: Útoky na dodavatelský řetězec softwaru, které cílí na vývojářské nástroje a repozitáře, jsou na vzestupu a představují jedno z největších rizik pro integritu vašeho kódu a bezpečnost dat.

Jak: Používejte nástroje pro skenování zranitelností v závislostech (SCA), ověřujte digitální podpisy balíčků, omezte oprávnění pro instalaci nových nástrojů a pravidelně auditujte používané komponenty. Zvažte také použití "supply chain security" řešení.
