SUBJECT: Kritické zranitelnosti a útoky na dodavatelský řetězec

🌍 Svět

Microsoft Defender zero-day zranitelnosti — Microsoft vydal bezpečnostní záplaty pro dvě zranitelnosti typu zero-day v nástroji Defender, které byly aktivně zneužívány v útocích. Je klíčové co nejdříve aplikovat aktualizace, aby se zabránilo potenciálnímu zneužití a ochránily systémy před známými hrozbami. Útok na dodavatelský řetězec vedl k průniku do GitHubu a Grafany — GitHub potvrdil, že průnik do 3 800 interních repozitářů byl způsoben kompromitací zaměstnaneckého zařízení prostřednictvím škodlivé verze rozšíření Nx Console pro VS Code, které bylo součástí širšího útoku na dodavatelský řetězec npm balíčku TanStack. Podobný incident postihl i Grafanu, kde útočníci zneužili neotočený GitHub workflow token, což zdůrazňuje rizika spojená s externími závislostmi a správou přístupových tokenů. [Zdroj: BleepingComputer, The Hacker News] URL: https://www.bleepingcomputer.com/news/security/github-links-repo-breach-to-tanstack-npm-supply-chain-attack/ URL: https://thehackernews.com/2026/05/github-internal-repositories-breached.html URL: https://www.bleepingcomputer.com/news/security/grafana-breach-caused-by-missed-token-rotation-after-tanstack-attack/

Kritická zranitelnost v Drupal Core — Drupal vydal bezpečnostní aktualizace pro "vysoce kritickou" zranitelnost v Drupal Core, která by mohla útočníkům umožnit vzdálené spuštění kódu (RCE), eskalaci oprávnění nebo získání informací, zejména na PostgreSQL databázích. Vzhledem k vysokému riziku zneužití se doporučuje okamžitá aktualizace všech instalací Drupalu. [Zdroj: The Hacker News, BleepingComputer] URL: https://thehackernews.com/2026/05/highly-critical-drupal-core-flaw.html URL: https://www.bleepingcomputer.com/news/security/drupal-critical-update-to-fix-bug-with-high-exploitation-risk/

Devět let stará zranitelnost v Linuxovém jádře a exploit PinTheft — Výzkumníci odhalili detaily zranitelnosti v Linuxovém jádře (CVE-2026-46333), která zůstala neodhalena devět let a umožňuje lokální eskalaci oprávnění. Navíc byl zveřejněn exploit pro nedávno opravenou zranitelnost PinTheft, která umožňuje lokálním útočníkům získat root oprávnění na systémech Arch Linux, což podtrhuje důležitost pravidelného patch managementu i u starších zranitelností. [Zdroj: The Hacker News, BleepingComputer] URL: https://thehackernews.com/2026/05/9-year-old-linux-kernel-flaw-enables.html URL: https://www.bleepingcomputer.com/news/linux/exploit-released-for-new-pintheft-arch-linux-root-escalation-flaw/

Obcházení MFA na SonicWall VPN kvůli neúplným záplatám — Útočníci obcházeli vícefaktorové ověřování (MFA) na zařízeních SonicWall Gen6 SSL-VPN, aby nasadili nástroje používané při ransomware útocích. To bylo možné kvůli neúplným záplatám, které ponechaly zadní vrátka otevřená, což zdůrazňuje nutnost důkladného ověřování a aplikace všech bezpečnostních záplat. [Zdroj: BleepingComputer] URL: https://www.bleepingcomputer.com/news/security/hackers-bypass-sonicwall-vpn-mfa-due-to-incomplete-patching/

⚠️ Příběh z praxe

Nedávné průniky do GitHubu a Grafany ukázaly, jak zranitelný může být dodavatelský řetězec softwaru. Útočníci kompromitovali populární vývojové nástroje a balíčky (rozšíření VS Code Nx Console, npm balíček TanStack), které obsahovaly škodlivý kód. Zaměstnanci, kteří si tyto nástroje stáhli, nevědomky otevřeli útočníkům dveře do interních systémů. Oběti zanedbaly důkladnou kontrolu softwaru třetích stran a nedostatečně rotovaly API tokeny po předchozích incidentech. Měly mít zavedeny přísnější politiky pro instalaci softwaru, pravidelnou rotaci všech citlivých klíčů a tokenů a robustní monitorování dodavatelského řetězce.

🛡️ Praktický tip

Co udělat: Důkladně prověřujte software a komponenty třetích stran, které používáte ve vašem vývojovém a provozním prostředí, a zaveďte přísné procesy pro správu a rotaci API klíčů a tokenů.

Proč: Útoky na dodavatelský řetězec jsou stále sofistikovanější a mohou vést k rozsáhlým průnikům, jako tomu bylo u GitHubu a Grafany. Neúplné záplatování a nedostatečná správa tokenů navíc otevírají dveře k obcházení MFA.

Jak: Používejte nástroje pro analýzu závislostí (SCA), prověřujte reputaci a integritu balíčků a rozšíření před jejich nasazením. Implementujte automatizovanou rotaci API klíčů a tokenů a vynucujte princip nejmenších privilegií. Pravidelně kontrolujte stav záplatování všech systémů, včetně VPN bran.

📋 Shrnutí a výzva k akci

Dnešní zprávy jasně ukazují, že útočníci se zaměřují na slabiny v dodavatelském řetězci softwaru, zneužívají kritické zranitelnosti v široce používaných systémech a hledají cesty k obcházení bezpečnostních kontrol, jako je MFA. Proaktivní přístup k zabezpečení je nezbytný pro ochranu vašich digitálních aktiv.

👉 Doporučená akce: Okamžitě zkontrolujte a aplikujte všechny dostupné bezpečnostní aktualizace pro operační systémy (zejména Linux), populární aplikace (jako Drupal) a bezpečnostní nástroje (jako Microsoft Defender a SonicWall VPN), a posilujte zabezpečení dodavatelského řetězce softwaru.
