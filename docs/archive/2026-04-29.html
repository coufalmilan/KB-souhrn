🌍 Svět

Útočníci zneužili kritickou zranitelnost v LiteLLM do 36 hodin — Bezpečnostní chyba (CVE-2026-42208) v populární Python knihovně LiteLLM, která slouží jako brána k různým AI modelům, byla aktivně zneužita pouhých 36 hodin po jejím zveřejnění. Zranitelnost typu SQL injection umožňuje útočníkům bez ověření vzdáleně spustit kód a získat přístup k citlivým datům, jako jsou API klíče. Tento incident podtrhuje nutnost extrémně rychlé reakce na nově objevené kritické zranitelnosti. [Zdroj: The Hacker News] URL: https://thehackernews.com/2026/04/litellm-cve-2026-42208-sql-injection.html

CISA nařídila záplatovat aktivně zneužívané chyby v ConnectWise a Windows — Americká agentura CISA přidala na svůj seznam známých zneužívaných zranitelností (KEV) dvě nové položky. Jedná se o kritickou chybu v nástroji pro vzdálenou správu ConnectWise ScreenConnect a zranitelnost ve Windows Streaming Service. Zařazení na seznam znamená, že obě chyby jsou aktivně využívány při útocích a federální úřady USA mají povinnost je neprodleně opravit, což je silným doporučením i pro všechny ostatní organizace. [Zdroj: The Hacker News] URL: https://thehackernews.com/2026/04/cisa-adds-actively-exploited.html

Nový ransomware VECT 2.0 se chová jako wiper a ničí data — Analýza ransomwaru VECT 2.0 odhalila zásadní chybu v jeho šifrovacím mechanismu. V důsledku této chyby ransomware nevratně poškozuje a ničí všechny soubory větší než 131 KB, místo aby je zašifroval. Pro oběti to znamená, že i po zaplacení výkupného není možné data obnovit. Jedinou ochranou jsou tak funkční a odpojené zálohy. [Zdroj: BleepingComputer] URL: https://www.bleepingcomputer.com/news/security/broken-vect-20-ransomware-acts-as-a-data-wiper-for-large-files/

Kritická zranitelnost v GitHubu umožňovala spuštění kódu — Byla objevena a opravena kritická zranitelnost (CVE-2026-3854) v platformách GitHub.com a GitHub Enterprise Server. Chyba mohla ověřenému útočníkovi umožnit vzdálené spuštění kódu (RCE) prostřednictvím jediného příkazu `git push`. Ačkoliv neexistují důkazy o jejím zneužití, doporučuje se všem správcům on-premise instancí GitHub Enterprise Server neprodleně aktualizovat. [Zdroj: The Hacker News] URL: https://thehackernews.com/2026/04/researchers-discover-critical-github.html

⚠️ Příběh z praxe

Společnost nasadila open-source nástroj LiteLLM pro zjednodušení práce s AI modely, aby urychlila vývoj. Útočníci však během pouhých 36 hodin od zveřejnění kritické zranitelnosti v této knihovně napadli jejich systémy a ukradli citlivé API klíče k placeným službám. Firma zanedbala aktivní monitoring a proces pro nouzové nasazení záplat pro komponenty třetích stran. Měla mít zavedené nástroje pro analýzu softwarových komponent (SCA) a připravený plán pro okamžitou aplikaci kritických oprav, jakmile jsou dostupné.

🛡️ Praktický tip

Co udělat: Zaveďte prioritní patch management pro aktivně zneužívané zranitelnosti.

Proč: Dnešní zprávy ukazují, že útočníci zneužívají nově zveřejněné zranitelnosti během hodin, nikoliv dnů. Čekání na běžný měsíční patchovací cyklus je pro kritické systémy příliš riskantní.

Jak: Pravidelně sledujte katalog CISA KEV (Known Exploited Vulnerabilities). Jakmile se v něm objeví software, který používáte, přidělte aplikaci záplaty nejvyšší prioritu a nasaďte ji mimo standardní cyklus, ideálně do 24-48 hodin.

Dnešní přehled ukazuje dramatické zkrácení doby mezi zveřejněním zranitelnosti a jejím masivním zneužitím. Klíčové je proto okamžitě reagovat na varování, jako je zařazení chyb do katalogu CISA KEV, a mít připravené procesy pro nouzové aktualizace.
